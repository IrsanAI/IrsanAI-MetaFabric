# MetaFabric Roadmap

This roadmap is the next continuation step after the command layer and the bilingual landing page.

---

## Phase 1 — Core Foundation

| Item | Goal | Output |
|---|---|---|
| Landing page | Make the product immediately understandable | Public-safe GitHub Pages entry |
| Command layer | Make SAMC, MFC, and SMC unambiguous | `IRSANAI Commands.md` |
| Feedback layer | Capture user signals early | Lightweight feedback area |

---

## Phase 2 — MetaFabric Control Plane

| Layer | Purpose | First artifact |
|---|---|---|
| Identity | User login and consent | OIDC / OAuth plan |
| Connector registry | Provider and self-hosted endpoints | JSON registry |
| Policy layer | Access, routing, and safety rules | Policy schema |
| Reflection layer | Explainability and confidence | Decision trace |

---

## Phase 3 — Expansion Paths

| Path | Why it matters | Notes |
|---|---|---|
| Self-hosted first | Privacy and sovereignty | Local / private endpoints |
| Provider routing | Interchangeable execution | Cloud model abstraction |
| Feedback analytics | Product learning | Anonymized trends only |
| Multilingual UX | Wider adoption | DE | EN first, more later |

---

## MFC Continuation Principle

MFC means:
- keep the core clean,
- add only what improves architecture,
- preserve the current system state,
- continue with causal consistency,
- avoid unnecessary scope expansion.
